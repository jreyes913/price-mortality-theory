\begin{abstract}
This version rolls in \emph{all} checked-in run artifacts: PMT grid sweep, AR(1)-GARCH(1,1), and the PMT+GARCH combination benchmark over the same S\&P 500 snapshot (2019-01-01 to 2025-01-01). Train strength is still strong for PMT and strongest for the combo build (combo train median up to 0.509 by 42-day horizon). But out-of-sample is where the truth hits: PMT leads GARCH on test medians at 10--31 days, GARCH leads PMT on test means at every horizon and clearly at 42 days, and the current combo run underperforms both on test means at every horizon and turns negative by 31/42 days. Straight read: PMT still has signal, but it is horizon-sensitive; GARCH is more stable on cross-sectional means; and this combo implementation is not production-ready yet.
\end{abstract}

\section{Introduction}
Vol forecasting is still hard, and the market does not care about in-sample flexing. PMT is built to read structural trend decay from price alone through a multiscale surface, while GARCH is the baseline most desks already trust. So the only fair move is full train/test separation, same universe, same date range, then compare outputs cleanly.

Earlier drafts leaned on PMT's train wins. This rewrite keeps it real: all three model families are included, and conclusions are driven by out-of-sample behavior first.

\section{Methods}
Define the Simple Moving Average field $SMA(W,t)$ on closing price $P_t$. PMT ``mortality'' is:
\begin{equation}
    \mu(W, t) = -\frac{1}{SMA} \left[ \frac{\partial SMA}{\partial W} + \frac{\partial SMA}{\partial t} \right]
\end{equation}
Finite differences approximate derivatives, and $\mu$ is lagged one period for causal evaluation.

Data and evaluation setup:
\begin{itemize}
    \item Universe: frozen S\&P 500 list in `data/sp500_tickers.txt`.
    \item Range: 2019-01-01 to 2025-01-01.
    \item Split: chronological 70/30 train/test.
    \item PMT sweep: $W \in [20,300]$ (step 20), $h \in [5,60]$ (step 5), best $(W^*,h^*)$ chosen on train.
    \item Horizon benchmark set: $h=\{10,21,31,42\}$ for PMT-vs-GARCH and combo comparisons.
\end{itemize}

\section{Results}
\subsection{Coverage and run integrity}
The latest checked-in manifests show 503 attempted tickers for each pipeline, with 497 PMT successes, 499 GARCH successes, and 500 combo successes. Failure logs are present for each run, so skipped names are auditable.

\subsection{PMT full-grid (best-parameter) behavior}
Across 497 PMT successes, in-sample quality remains strong (mean $\rho_{train}=0.508$, median $0.531$). Out-of-sample for those train-selected best parameters compresses sharply (mean $\rho_{test}=-0.018$, median $0.010$; 489 non-null test observations).

Parameter selection remains concentrated at the edge of the search grid: $W^*$ mean 241.4 (median 300) and $h^*$ mean 43.0 (median 60). In plain terms: train optimization keeps pushing toward longer windows/horizons, but that preference does not transfer reliably to test.

\subsection{Fixed-horizon PMT vs GARCH}
At fixed horizons, PMT and GARCH tell a split story (Table~\ref{tab:pmt-garch}):
\begin{itemize}
    \item PMT has stronger \emph{test median} at 10, 21, and 31 days.
    \item GARCH has stronger \emph{test mean} at every horizon.
    \item By 42 days, PMT mean/median are near zero to negative while GARCH remains positive.
\end{itemize}

\subsection{Combination run (PMT + GARCH)}
The combo run was expected to improve robustness; right now it does not. It is very strong in-sample (train medians 0.463, 0.481, 0.498, 0.509 from 10 to 42 days), but test performance drops hard:
\begin{itemize}
    \item Test means: 0.053 (10d), 0.029 (21d), -0.012 (31d), -0.040 (42d).
    \item Test medians: 0.059 (10d), 0.036 (21d), -0.012 (31d), -0.039 (42d).
\end{itemize}

Against PMT and GARCH in matched observations, combo wins only about 29.6\% vs PMT and 43.8\% vs GARCH on test $\rho$. So this blend, as implemented now, is overfit on train and not delivering out-of-sample lift.

\section{Discussion}
Professional summary, no hype:
\begin{itemize}
    \item PMT is still useful, especially for short-to-mid horizons when judged by medians (cross-sectional robustness proxy).
    \item GARCH is the steadier choice on test means and becomes clearly better at longer horizons.
    \item The current combo stack is not ready; train gains are not surviving test.
\end{itemize}

So the deployment logic is simple: keep PMT in the feature set, keep GARCH as a backbone baseline, and treat combo architecture as an active research track rather than a shipped model. Next upgrades should prioritize regularization, stricter walk-forward validation, and horizon-specific weighting instead of static blending.

\section{Conclusion}
All-in evidence says PMT has real signal but not universal dominance. Horizon sensitivity is the key fact. GARCH remains competitive to superior on mean out-of-sample behavior, and the present PMT+GARCH combo run fails to add test alpha. The path forward is disciplined: horizon-aware calibration, anti-overfit controls, and ensemble design that is validated out-of-sample before any production claims.
